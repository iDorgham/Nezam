# Design catalog (source profiles)

**Purpose:** `.cursor/design/` is the **pick list** of visual-system profiles. Each profile is exactly:

`.cursor/design/<brand>/design.md`

**Project contract (what implementation follows):** after you choose a profile, copy it to the repository root as **`DESIGN.md`**. That file is the SDD design authority for this repo (see `.cursor/rules/workspace-orchestration.mdc`: preferred path is root `DESIGN.md`; `docs/DESIGN.md` is legacy-compatible only if you mirror it yourself).

**Apply a profile to the project**

- **Command:** `/START design` in Cursor (browse, confirm, then copy to root `DESIGN.md`), or
- **Script:** `pnpm run design:apply -- <brand>`  
  (runs `scripts/design/copy-profile-to-design-md.sh`).

**After copying:** note the active brand in `docs/workspace/context/CONTEXT.md` (working notes) if helpful for handoffs; do not treat catalog files under `.cursor/design/` as the live contract—treat root `DESIGN.md` as canonical once selected.
