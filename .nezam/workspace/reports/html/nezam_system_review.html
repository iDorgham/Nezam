Honest Assessment — Strengths vs Weaknesses

✅ Strengths

Product-Type Intelligence
Auto-detects product type and runs the correct pipeline order — Website ≠ SaaS ≠ Mobile. Competitors treat all projects the same.
Hardlock System
Real enforcement, not suggestions. Can't enter BUILD without DESIGN.md. Can't enter DESIGN without IA. Prevents the #1 reason AI-assisted projects fail.
Design Gates (7 gates)
Token-first, fluid typography, motion budget, 3D fallback, component API, perf thresholds — all enforced before a single line of code.
Multi-Tool Orchestration
Gemini for docs, Qwen for Arabic, Kilocode for scans, OpenCode for boilerplate — silently, without user intervention. 60-80% token savings.
MENA / Arabic-First
RTL specialists, Khaleeji/Levantine/Masri/MSA dialect agents, Arabic SEO/AEO. No other method has this.
Rollback Protocol
Partial rollback, full replan, and amendment paths are defined and documented. Most methods silently drift.

❌ Weaknesses

Context Window Pressure
100+ agents + rules always-apply = heavy token load at session start. The compression protocol exists but isn't automatic enough.
No Runtime Agent Bus
Agent communication is simulated through documents. There's no real inter-agent message passing — agents don't actually talk to each other in real-time.
Drift Between .cursor/ and Mirrors
Wave 2 wiring audit found .mdc rule files not fully updated to match command contracts. pnpm ai:sync exists but isn't enforced on every change.
Wireframe Layer is Thin
wireframe-to-spec-converter exists as a skill but isn't connected to a real wireframe tool (Figma/Excalidraw/Penpot MCP).
No Spec Versioning
SPEC.md files have amendment logs but no semantic version tracking per spec. Hard to track what version of a spec was built.
PROGRESS_REPORT is a Template
Still showing placeholder content. Memory and progress tracking are architecturally sound but not populated for this project.

Where Kiro/SpecKit Beat NEZAM

Kiro's Advantage
Steering specs are file-native — they hook into the IDE and auto-trigger on file changes, not just slash commands.
Visual wireframe pipeline — Kiro connects specs to real UI previews before code.
Smaller, tighter context — Kiro agents are scoped narrow; NEZAM's breadth creates pressure.
SpecKit's Advantage
Spec versioning is first-class — every spec has a version, changelog, and diff. NEZAM tracks amendments but not spec versions per se.
Simpler agent model — fewer agents = easier for solo founders to navigate and understand what's running.
Spec → code traceability — SpecKit can show which spec line generated which code line. NEZAM does this via SPEC.md but not automated.
