workspace-orchestration.mdc

alwaysApply: true

Defines pipeline order (never skip knowingly). Enforces SDD. Sets memory layer model. Mandates Recommendation footer on every substantive reply. Governs cross-client sync (canonical = .cursor/, generated = everything else).

design-dev-gates.mdc

alwaysApply: true

Hard-blocks /DEVELOP until all 7 design quality gates pass. No bypasses.

G1
Token-First CSS
Zero hardcoded px/hex/rem. All primitives through governed tokens.
G2
Fluid Type & Grid
clamp() scales, container queries, explicit breakpoint matrix.
G3
Animation Budget
transform/opacity only. prefers-reduced-motion mandatory.
G4
Progressive 3D
Fallback chain: WebGL → SVG → static. Suspense boundaries required.
G5
Component API
Typed, variant-driven, forwardRef, tree-shakeable, Storybook-ready.
G6
Perf & A11y
LCP <2.5s · CLS <0.1 · INP <200ms · WCAG 2.2 AA.
G7
Design Alignment
All UI must trace to DESIGN.md. Drift without a decision record fails the gate.

sdd-design.mdc

conditional (docs/**)

Before any UI implementation: lock SEO artifacts → lock IA/menus → lock content map → author DESIGN.md → then code. Also enforces versioning discipline and conventional commits.

nezam-design-gates-pro.mdc

alwaysApply: true

Pro-grade extension of design-dev-gates. Identical 7-gate model with added requirement for multi-agent handoff evidence (nezam-multi-agent-handoff) and active subphase prompt artifact check (prompt.json + PROMPT.md).

ui-ux-swarm-library-ds-content.mdc

alwaysApply: true

Governs library/design-system/content swarm routing. Defines ContentSpecSchema with RTL-ready, a11y, and word-limit fields. Color sync must be token-driven with AA/AAA contrast validation. Each agent pass must return: diff · rationale · validation · next trigger.

workspace-client-onboarding-gate.mdc

conditional (workspaces/clients/**)

When working in client workspaces, reads .ai/onboarding/state.yaml first. Blocks /plan blueprint and /dev implement until onboarding_complete = true. Allows /guide help and /onboard commands at any time.

Memory Architecture — 4 Layers

Layer 1 — Source of Truth

Repository files. Committed specs, decisions, templates. Never ephemeral.

docs/core/required/** · templates · gate artifacts

Layer 2 — Working / Session

AI chat thread and open files. Ephemeral — persist outcomes into docs before session ends.

Cursor chat · open file buffers · agent working memory

Layer 3 — Durable Project Memory

Long-form project knowledge. Updated via /SAVE log.

docs/workspace/context/MEMORY.md · phase logs

Layer 4 — External Companion

Browser AI (Grok, Qwen, Gemini, Claude web). User pastes context + latest progress report.

docs/reports/progress/PROGRESS_REPORT.latest.md · CONTEXT.md
