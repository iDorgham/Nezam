# Commit Conventions by Sub-Phase

- `feat(seo):` for `01-content-foundation/01-research` and `01-content-foundation/02-strategy`  
Example: `feat(seo): define high-intent keyword clusters for onboarding guides`
- `feat(content):` for `01-content-foundation/03-create`  
Example: `feat(content): draft product listing and pricing page copy`
- `feat(legal):` for `01-content-foundation/04-compliance`  
Example: `feat(legal): add GDPR and cookie disclosure copy to legal draft`
- `feat(design):` for all of `02-design-system`  
Example: `feat(design): finalize spacing scale tokens and button states`
- `chore(ci):` for `03-build/01-env`  
Example: `chore(ci): configure lint, format, and test jobs in pipeline`
- `feat(backend):` for `03-build/02-backend`  
Example: `feat(backend): add authenticated CRUD endpoints for listings`
- `feat(ui):` for `03-build/03-frontend`  
Example: `feat(ui): implement route guards and dashboard navigation shell`
- `feat(features):` for `03-build/04-features`  
Example: `feat(features): integrate search and saved-items vertical slice`
- `fix(tests):` for `03-build/05-test`  
Example: `fix(tests): stabilize flaky E2E auth scenario and fixtures`
- `chore(release):` for `03-build/06-release`  
Example: `chore(release): generate changelog and run staging smoke suite`
- `fix(harden):` for `04-harden`  
Example: `fix(harden): reduce bundle blocking scripts and close CSP gaps`
- No strict prefix for `05-ship`; prefer `docs(release):` or similar  
Example: `docs(release): publish go-live notes and rollback playbook`
